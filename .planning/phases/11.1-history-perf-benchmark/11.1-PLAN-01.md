# Plan 11.1-01: History-Enabled Performance Benchmark

## Objective

Close PERF-02 gap: measure `trigger()` throughput with `enable_history()` active, assert ≤ 2× degradation vs. disabled baseline, and document measured overhead in README.

## Tasks

### Task 1: Add history-enabled throughput benchmark test

**File:** `tests/test_performance_benchmarks.py`

Add a new test `test_trigger_history_enabled_throughput` in the `TestPerformanceBenchmarks` class, immediately after `test_trigger_min_throughput`. The test should:

1. Create the same minimal 2-state toggle FSM used by `test_trigger_min_throughput`
2. Run a baseline measurement (history disabled) — 200k triggers, record ops/sec
3. Enable history with `enable_history(max_entries=1000)`
4. Run the same 200k triggers with history enabled, record ops/sec
5. Assert history-enabled throughput is no more than 2× slower than disabled baseline
6. Use the same compiled/pure-Python detection as the existing test
7. Print measured overhead for documentation purposes

### Task 2: Document history-enabled performance in README

**File:** `README.md`

Update the Performance Characteristics table to include a row for `trigger()` with history enabled, showing the measured overhead characteristic (e.g., "≤ 2× overhead" or similar).

Update the Transition History bullet point to mention measured performance impact.
