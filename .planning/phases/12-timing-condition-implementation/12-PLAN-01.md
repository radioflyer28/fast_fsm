# Plan 12-01: Timing Condition Implementation

## Objective

Implement three time-based condition classes — `TimeoutCondition`, `CooldownCondition`, and `ElapsedCondition` — in `condition_templates.py` and export them from `fast_fsm.__init__`. These are standard `Condition` subclasses that use `time.monotonic()` for platform-safe timing guards and follow the library's `__slots__` convention.

Purpose: Users should be able to express timing guards on FSM transitions without writing clock logic.
Output: Three new condition classes importable from `fast_fsm`, ready for use as transition guards.

**1. `TimeoutCondition` (TIME-01, TIME-04, TIME-05, TIME-06, TIME-07)**
```python
class TimeoutCondition(Condition):
    """Allows transitions until a timeout expires. Returns True before timeout, False after."""

    __slots__ = ("seconds", "_ref")

    def __init__(self, seconds: float):
        super().__init__(f"timeout_{seconds}", f"Blocks after {seconds}s")
        self.seconds = seconds
        self._ref = time.monotonic()

    def check(self, **kwargs) -> bool:
        return (time.monotonic() - self._ref) < self.seconds

    def reset(self) -> None:
        self._ref = time.monotonic()
```

**2. `CooldownCondition` (TIME-02, TIME-04, TIME-05, TIME-06, TIME-07)**
```python
class CooldownCondition(Condition):
    """Enforces a minimum interval between successful checks. Passes on first call, then blocks until cooldown elapses since last success."""

    __slots__ = ("seconds", "_last_success")

    def __init__(self, seconds: float):
        super().__init__(f"cooldown_{seconds}", f"Minimum {seconds}s between successes")
        self.seconds = seconds
        self._last_success: float = 0.0  # epoch 0 ensures first call always passes

    def check(self, **kwargs) -> bool:
        now = time.monotonic()
        if (now - self._last_success) >= self.seconds:
            self._last_success = now
            return True
        return False

    def reset(self) -> None:
        self._last_success = 0.0
```
Note: `_last_success` is initialised to `0.0` (monotonic epoch). Since `time.monotonic()` returns a value far greater than 0.0, the first `check()` always passes — no explicit "first call" flag needed.

**3. `ElapsedCondition` (TIME-03, TIME-04, TIME-05, TIME-06, TIME-07)**
```python
class ElapsedCondition(Condition):
    """Returns True once a minimum elapsed time has passed. Returns False before, True after."""

    __slots__ = ("seconds", "_ref")

    def __init__(self, seconds: float):
        super().__init__(f"elapsed_{seconds}", f"Passes after {seconds}s elapsed")
        self.seconds = seconds
        self._ref = time.monotonic()

    def check(self, **kwargs) -> bool:
        return (time.monotonic() - self._ref) >= self.seconds

    def reset(self) -> None:
        self._ref = time.monotonic()
```

Key implementation details:
- All classes use `__slots__` — no `__dict__` (TIME-07)
- All `check()` methods accept `**kwargs` (TIME-06)
- All use `time.monotonic()` exclusively — no `time.time()` (TIME-04)
- All provide `reset()` returning `None` (TIME-05)
- Single `time.monotonic()` call per `check()` invocation for hot-path performance
- Auto-generated `name` and `description` from class name + threshold per CONTEXT.md decisions
- Classes placed in `condition_templates.py` alongside existing helpers (INT-01)
