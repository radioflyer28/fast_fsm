# Phase 7: Serialization (`to_dict()`) — Plan 01

**Created:** 2026-04-05
**Requirements:** SERIAL-01, SERIAL-02, SERIAL-03

## Tasks

### Task 1: Implement `to_dict()` in `core.py`

**File:** `src/fast_fsm/core.py`
**Location:** After `from_dict()` classmethod (around line 530)

Add `to_dict(self) -> Dict[str, Any]` method that:
1. Returns `{"name": self._name, "initial": self._initial_state.name, "states": sorted(list(self._states.keys())), "transitions": [...]}`
2. Transitions: iterate `self._transitions` dict (structure: `{from_state_name: {trigger_name: TransitionEntry}}`) and produce `{"trigger": trigger, "from": from_state, "to": entry.to_state.name}` for each entry
3. Sort transitions for deterministic output — by `(from, trigger, to)` or similar
4. Guards/conditions are NOT included (per SERIAL-02) — `TransitionEntry.condition` is ignored

### Task 2: Add tests for `to_dict()` in `tests/test_advanced_functionality.py`

Add a new `TestToDict` class with tests:
- `test_basic_to_dict` — simple FSM with 2-3 states and transitions, verify returned dict has keys `name`, `initial`, `states`, `transitions` with correct values
- `test_to_dict_roundtrip` — build FSM, call `to_dict()`, pass result to `StateMachine.from_dict()`, verify reconstructed machine has identical state names, identical triggers, and same transition graph (trigger from same source goes to same target)
- `test_to_dict_with_guards_excludes_conditions` — add a guarded transition, verify `to_dict()` output contains the transition but no condition/guard keys
- `test_to_dict_with_multiple_from_states` — build FSM where `add_transition("reset", ["running", "error"], "idle")` fans out; verify `to_dict()` produces separate `{"trigger": "reset", "from": "running", ...}` and `{"trigger": "reset", "from": "error", ...}` entries
- `test_to_dict_json_serialisable` — `json.dumps(fsm.to_dict())` succeeds without `TypeError`
