## Unsatisfied Requirements

### PERF-02: History-enabled throughput not measured

**Requirement:** `trigger()` throughput with history **enabled** is measured and documented (no hard floor — observable degradation acceptable; must not be >50% slower than disabled)

**Current state:**
- History recording is implemented and tested functionally ✅
- Zero-cost when disabled via single `None` check ✅
- **Missing:** No benchmark test measures trigger() throughput WITH `enable_history()` active
- **Missing:** No documentation of actual overhead (README, docstring, or test comments)

**To satisfy:** Add a benchmark in `test_performance_benchmarks.py` that enables history, measures throughput, and asserts ≤ 2× degradation vs. disabled baseline. Document measured overhead in README or docstring.
